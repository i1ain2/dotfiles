# Ankiカード自動生成インストラクション

## 📖 概要

このドキュメントは、プログラミング・技術概念の学習を効率化するためのAnkiカード自動生成ガイドです。
提示されたお題に対して、カードテンプレートから適切なテンプレートを選んでAnkiカードを生成してください。

## 🎯 カードテンプレート

### 基礎を固める

#### 1. 概念説明カード
**用途**: 技術用語や概念の理解を定着させる

**形式**:
- Front: `[技術用語]とは何か？`
- Back: 定義 + 利点/制約 + 具体例

**例**:
```markdown
### カード: Rustの所有権

**Front:**
Rustの所有権（Ownership）とは何か？

**Back:**
メモリ安全性を保証するコンパイル時のルール。各値は1つの所有者を持ち、所有者がスコープを抜けると値は破棄される。

**利点**:
- メモリリーク防止
- データ競合防止
- GC不要

**制約**:
- 学習曲線が急
- コンパイラとの格闘

**例**:
```rust
let s = String::from("hello"); // sが所有者
```

### 出力フォーマット

各カードは以下のMarkdown形式で出力してください：

```markdown
### カード: [カードタイトル]

**Front:**
[質問文やプロンプト]

**Back:**
[回答・解説]
- コードブロックは標準的なMarkdown記法（```言語名）を使用
- 箇条書きや番号付きリストも標準的なMarkdown記法を使用
- 強調は **太字** や見出しを使用
```

## 💡 Tips

### カードの粒度
- 1枚のカードには1つの概念
- 複雑な内容は複数カードに分割
- コードは10行以内が理想
